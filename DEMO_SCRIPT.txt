[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's look at the underlying architecture. When a transaction query is submitted, it enters the START node and immediately goes through our Security Checkpoint. If a security issue is detected, it is immediately routed to the Security Event Handler, which blocks the execution. Otherwise, it proceeds along the 'safe' route to our main Orchestrator Agent. The Orchestrator manages two specialized sub-agents: the Transaction Analyzer, which parses transaction logs, and the Compliance Auditor, which checks for custom overrides and evaluates risk. Both sub-agents leverage our local Model Context Protocol Server to look up company policies and employee allowances. Once the sub-agents finish, the Orchestrator aggregates the results into a structured audit report. The Route Decision node evaluates this report: if it's a low-risk spend, it auto-approves and generates the Final Report; if it is high-risk, it routes to the Human Approval Node, which triggers a Human-in-the-Loop pause until a manager reviews it.
